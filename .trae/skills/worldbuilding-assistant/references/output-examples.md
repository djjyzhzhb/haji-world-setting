# 输出范例

> 此文件包含完整的"应该做什么 / 不应该做什么"对照范例，从 SKILL.md 正文中拆分以保持主文件精简。

---

## 范例 A：创建新地理区域

```
用户："帮我在大陆A的西北部创建一个高原地区"

AI 应执行：
  1. 读 _templates/template_region.md 获取结构
  2. 读 02_地理/05_大陆A/00_大陆A总览.md 了解大陆概况
  3. 读 02_地理/02_气候带与生态区.md 推演高原气候
  4. 读 02_地理/03_自然资源分布.md 确定资源
  5. 按"地形 → 气候 → 资源 → 人口承载力 → 战略价值"组织输出
  6. 标注 causal_refs 指向依赖的文档

AI 不应做：
  ✗ 直接套用青藏高原/安第斯模板
  ✗ 不查大陆A总览就假设西北部的地质构造
  ✗ 跳过经济基础直接赋予"贸易枢纽"地位
```

## 范例 B：生成叙事场景

```
用户："写一段场景：一个种族A的年轻猎人在冬至日独自进入森林"

AI 应执行：
  1. 读 04_种族与生物/02_种族A/ 了解种族A的生理/文化/历史
  2. 读 01_世界核心/04_时间体系.md 确认"冬至"在该世界的历法含义
  3. 读 02_地理/ 确认该种族生活区域的气候和森林生态
  4. 用限制性第三人称，只通过猎人的感官呈现世界
  5. 对话/内心独白需反映该角色的年龄、职业、宗族背景

AI 不应做：
  ✗ 让猎人说出现代生态学知识
  ✗ 内心独白中直接解释种族A的历史
  ✗ 描写不存在的动物/植物
```

## 范例 C：从 _inbox/ 素材转化设定

```
用户："把 _inbox/人造语言粗略评估报告.md 里关于哈吉语使用者的描述转成种族A的文化设定"

AI 应执行：
  1. 读 _inbox/人造语言粗略评估报告.md 提取关于使用者的段落
  2. 确认 04_种族与生物/02_种族A/02_文化与习俗.md 的当前状态
  3. 筛选与"文化习俗""社会结构""命名传统"相关的内容
  4. 按 Mimesis 链路验证转化后的设定
  5. 在目标文档中标注来源："素材来自 _inbox/人造语言粗略评估报告.md"

AI 不应做：
  ✗ 直接复制 _inbox 原文（需转化）
  ✗ 跳过现有设定的冲突检查
  ✗ 删除 _inbox 原始文件
```

## 范例 D：修改已有设定并追溯影响

```
用户："把种族A的平均寿命从 80 岁改成 200 岁"

AI 应执行：
  1. 读 04_种族与生物/02_种族A/01_生理与特性.md，定位寿命描述
  2. 搜索所有引用该文档的 causal_refs（反向索引）
  3. 检查影响链：寿命变长 → 人口结构变化 → 经济/社会/文化连锁反应
  4. 列出一份"需要联动修改的文档清单"给用户确认
  5. 确认后逐个修改，每改一份都更新版本日志

AI 不应做：
  ✗ 只改生理文档那一个数字
  ✗ 不检查历史年表中是否引用了"80 岁的寿命"
  ✗ 不追问"为什么会活 200 年"（必须先有原因，才能设后果）
```
