## 二、不可妥协的三条核心哲学

**1. Mimesis（广义现实主义）**：一切设定必须符合因果律，有物质基础和历史条件支撑。

**2. 语言不在此定义**：哈吉语的音系、语法、词汇在外部项目，本项目只负责其文化土壤。

**3. 世界先于人物**：角色是环境的产物，不可脱离世界设定凭空创造。

---

## 三、物质条件决定论（Mimesis 的执行引擎）

任何设定点必须按此链条逐级验证，不可跳级：

```
地理 → 经济 → 社会 → 文化 → [力量体系] → 个体心理
```

**为什么这个顺序不可打破**：地理决定资源分布，资源决定经济模式，经济决定社会结构，社会结构塑造文化习惯，文化习惯影响个体心理。倒过来就是空中楼阁。

每级追问映射表见 `references/file-index.md` 第二节。

> 力量体系附加验证（见 `_consistency_rules.md` 第 1.3 节和 `13_因果律与一致性/01_核心理论/03_力量体系与社会的互动.md`）：
> - 力量有来源（可追溯，不能是"天生的天赋"）
> - 力量有代价（经济/生理/社会，至少一项）
> - 力量有社会影响（持有者如何被对待？社会结构因此如何改变？）

## 四、标准操作流程

### 4.1 创建新设定文档

1. 选模板 → `_templates/`
2. 读约束 → `_consistency_rules.md` 相关条款
3. 填因果 → `起因 → 表现 → 后果 → 与其他设定的因果联系`
4. 验物质 → 按第三节链条逐级验证
5. 查术语 → 对齐 `_glossary.md`；新术语写入 glossary
6. 标关键 → `> [AI关键] 具体约束内容`
7. 自检 → 执行第七节的反馈循环

### 4.2 修改现有文档

1. 查因果联系 → 读取 `causal_refs` 中的引用文档
2. 追溯影响 → 搜索所有引用被修改设定的文档，列出影响清单
3. 记录矛盾 → 按 `_consistency_rules.md` 矛盾处理流程操作
4. 更新元数据 → 修改 `modified` 日期，必要时调整 `status`

### 4.3 生成叙事内容

**可判定标准**（每条附判定方法）：

| 要求 | 判定方法 |
|------|---------|
| 无信息倾销 | 逐段问：角色 A 对 B 说的信息，B 本来知道吗？如果 B 不需要知道且对话的唯一作用是向读者解释设定 → 违规，删除 |
| 无跨角色内心独白 | 扫描：同一场景中是否切换了两个以上角色的内心视角？有 → 违规 |
| 语体差异可辨识 | 遮住名字重读对话，能根据说话方式分辨是谁 → 通过。分辨不出 → 修改 |
| 时空校准 | 历法（`01_世界核心/04_时间体系.md`）和地理描述一致 |

### 4.4 叙事输出

- **位置**：`故事/` 目录
- **命名**：`故事/[种族或主题]-[场景关键词].md`
- **Front Matter**：`category: "narrative"`，`status: draft`

### 4.5 `_inbox/` 素材处理

1. 记录原始文件名和导入日期
2. 格式转换（docx/pdf → Markdown）
3. 只提取与本项目设定相关的部分
4. 原始文件留在 `_inbox/`，不删除
5. 目标文档中标注来源：`素材来自 _inbox/原始文件名`

输出范例见 `references/output-examples.md`。

### 4.6 版本记录

每次操作后：
1. 更新文档 `modified` 日期
2. 在 `00_项目总览/02_版本日志.md` 追加：`YYYY-MM-DD: [新建/修改/素材导入/矛盾修正] 文件路径 — 简述`

## 六、文风红线

> 完整规范见 `_style_guide.md`。

1. ❌ 评价性形容词（"伟大""极其强大""卓越"）→ 精确描述替代
2. ❌ 现实文化比喻（"像罗马帝国一样"）→ 不得出现
3. ❌ 角色跨认知发言 → 角色不能说出超越其时代/教育/职业认知的话

**违规处理**：删除违规段落，重新生成。完成后对照 `_style_guide.md` 全文复查。

## 七、一致性自检（反馈循环）

> 完整清单见 `13_因果律与一致性/02_实践工具/02_一致性检查清单.md`

生成设定内容后，执行以下循环直到全部通过：

```
逐项自检 → 发现问题 → 修复 → 重新从头自检 → 8 项全通过 → 完成
```

| # | 检查项 | 不通过时做什么 |
|---|--------|--------------|
| 1 | 有前因 ✓ | 补充可追溯的起因 |
| 2 | 有后果 ✓ | 展开连锁反应 |
| 3 | 物质条件支撑 ✓ | 按第三节链条推导 |
| 4 | 力量体系未越界 ✓ | 有无代价使用 → 修复 |
| 5 | 术语统一 ✓ | 对齐 `_glossary.md` |
| 6 | 时间一致 ✓ | 查 `04_时间体系.md` |
| 7 | 风格一致 ✓ | 对照 `_style_guide.md` |
| 8 | 无矛盾 ✓ | 搜索相关文档 |

## 八、哈吉语工具

### 文档嵌入格式

**逐词注音**：`{{ruby: mogu}}` → 系统自动分词为 `mo` `gu`，标注汉字和 PUA 字符。

**整句混排**：`((((我来到 mogu naxomi 山下。))))` → 仅对拉丁字母注音。

### 新增哈吉语内容

1. 确认词汇不与现有对照表重复
2. 按构词法造词（CVC 词干 + 后缀如 `-koyi`、`-r`）
3. `ruby.ts` 确认 `tokenize/toHan/toPua` 可处理
4. 使用上述标记嵌入文档
5. `ruby-demo.html` 中测试渲染

## 九、常见陷阱

| 不要做 | 因为 | 正确做法 |
|--------|------|---------|
| 先写角色再补世界 | 违反第三条核心哲学 | 先查/写世界设定 |
| 凭空创造技术 | 违反第三节物质链条 | 按链条逐级验证 |
| 力量无代价 | 违反 `_consistency_rules.md` 1.3 | 经济/社会/个人代价至少一项 |
| 术语前后不一致 | 产生矛盾 | 以 `_glossary.md` 为准 |
| "因为剧情需要" | 违反因果律 | 可追溯的起因 |
| 不确定时硬填 | 制造虚假一致性 | 标"待定"留白 |
