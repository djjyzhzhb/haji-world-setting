# 手机端 UI 适配 Spec

## Why
当前项目仅有 2 个简单的响应式断点（768px / 600px），仅调整了侧边栏宽度和内容区内边距。在手机端（320px–428px 视口）存在大量可用性问题：侧边栏占用过多空间且无法隐藏、标注面板和变更面板过窄、触控目标过小、主页标题过大、表格溢出等。

## What Changes
- 侧边栏在手机端改为可滑出的抽屉式导航，默认隐藏，通过汉堡按钮触发
- 内容区域字体和间距在手机端等比缩小
- 标注面板（annotate panel）在手机端改为全屏，增大触控目标
- 变更记录面板（changes panel）在手机端全宽显示，增大触控目标
- 角标（badge）在手机端略微放大以适配手指触控
- 气泡弹窗（popover）在手机端限制最大宽度避免溢出
- 主页标题在手机端缩小字号
- 表格包裹横向滚动容器
- 格式化工具栏按钮间距适配触控

## Impact
- Affected specs: 无（新 spec）
- Affected code: `web/src/style.css`（主要）、`web/src/main.ts`（汉堡按钮逻辑）、`web/src/ui/changes.ts`（面板行为）

## ADDED Requirements

### Requirement: 手机端侧边栏抽屉式导航
系统 SHALL 在视口宽度 ≤ 768px 时将侧边栏改为覆盖式抽屉，默认隐藏，通过顶部汉堡按钮切换显示/隐藏。

#### Scenario: 手机端侧边栏默认隐藏
- **WHEN** 用户在宽度 ≤ 768px 的设备上打开页面
- **THEN** 侧边栏不占据布局空间，页面顶部显示汉堡按钮（☰），内容区占满全宽

#### Scenario: 点击汉堡按钮展开侧边栏
- **WHEN** 用户点击汉堡按钮
- **THEN** 侧边栏从左侧滑入，覆盖在内容区上方，同时显示半透明遮罩层

#### Scenario: 点击遮罩或导航项关闭侧边栏
- **WHEN** 用户点击遮罩层或选择任意导航项
- **THEN** 侧边栏滑出隐藏，遮罩消失

### Requirement: 手机端内容区自适应
系统 SHALL 在视口宽度 ≤ 768px 时调整内容区排版，使其可读且不横向溢出。

#### Scenario: 内容区内边距和字号缩小
- **WHEN** 用户在手机端查看文档内容
- **THEN** 内容区左右内边距为 16px，正文字号 15px，标题字号等比缩小

#### Scenario: 表格横向滚动
- **WHEN** 文档内容包含宽表格且视口不足以完整显示
- **THEN** 表格容器出现横向滚动条，表格本身不溢出视口

### Requirement: 手机端标注面板全屏化
系统 SHALL 在视口宽度 ≤ 600px 时将标注面板（annotate panel）改为接近全屏的模态框。

#### Scenario: 手机端打开标注面板
- **WHEN** 用户在手机端点击"+"按钮打开标注面板
- **THEN** 面板卡片宽度为 100vw，高度为 100vh（或 100dvh），边角无圆角或极小圆角，表单字段和按钮触控目标 ≥ 44px

#### Scenario: 格式化工具栏在手机端不换行拥挤
- **WHEN** 标注面板在手机端打开
- **THEN** 格式化工具栏按钮间距适当，所有 6 个按钮在一行内可见，按钮最小尺寸 40×36px

### Requirement: 手机端变更记录面板全宽化
系统 SHALL 在视口宽度 ≤ 600px 时将变更记录面板（changes panel）改为全宽。

#### Scenario: 手机端打开变更记录面板
- **WHEN** 用户在手机端点击右下角编辑图标打开变更面板
- **THEN** 面板宽度为 100vw，从右侧滑入，操作按钮间距增大以适配手指

### Requirement: 手机端角标放大
系统 SHALL 在视口宽度 ≤ 600px 时将角标（badge）略微放大以适配手指触控。

#### Scenario: 手机端角标尺寸
- **WHEN** 用户在手机端查看有角标的段落
- **THEN** 角标最小尺寸为 26×26px，字号 13px，确保手指可点击

### Requirement: 手机端气泡弹窗宽度限制
系统 SHALL 在视口宽度 ≤ 600px 时限制气泡弹窗（popover）最大宽度，避免溢出。

#### Scenario: 手机端气泡弹窗
- **WHEN** 用户在手机端点击角标
- **THEN** 气泡弹窗最大宽度为 90vw，内部文字不溢出

### Requirement: 手机端主页标题缩小
系统 SHALL 在视口宽度 ≤ 600px 时将主页标题字号缩小。

#### Scenario: 手机端主页标题
- **WHEN** 用户在手机端查看主页（home.md）
- **THEN** h1 字号为 32px（桌面端 48px），正文 16px，margin-top 调整适配
