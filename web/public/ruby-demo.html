哈吉文 · Ruby 注音Haji Ruby Annotation

哈吉文通过 <ruby> 元素标注在汉字上方，类似日文振假名。 用户无需安装任何字体或输入法——字体文件随网页自动加载。

一、人名示例

女性名 · 姓 + 名 + 称
猫古 · 娜笑米 · 桥可
逐字对应：mo→猫 gu→古 na→娜 xo→笑 mi→米 qo→桥 ko→可

哈吉语
hajiko 中「语」为中文意译，仅固有词 haji（哈吉）整体注音

三、HTML 标记方法

<ruby>猫<rt>&#xE326;</rt></ruby><ruby>古<rt>&#xE22C;</rt></ruby>

翻译字与哈吉音节一一对应时，逐字标注最整齐。若不对齐（如三音节译成两字），则用一个 <ruby> 包裹整个词：<ruby>猫古<rt>音节串</rt></ruby>

配合 CSS 全局生效：

@font-face {
  font-family: 'HaJi';
  src: url('哈吉文.woff2') format('woff2');
  unicode-range: U+E000-E5FF;  /* 只影响 PUA 区 */
}

rt { font-family: 'HaJi', serif; }

unicode-range 让字体只作用于哈吉文码点段，不影响页面其他文字。

四、在设定文档里直接写哈吉文

打开项目里任意 .md 文档后，你可以用下面三种标记嵌入哈吉文，无需跳转到此工具页。
渲染由主页的 replaceRubyMarkers 逻辑处理，打开文档时自动生效。

1. 整句/段落级混写 (((( ... ))))

写法：用四对括号包裹一段文字，只有里面的拉丁字母会被当作哈吉语做 ruby 注音，中文、标点、空格全部原样保留。

((((哈吉人姓名分三部分：mogu（姓）· naxomi（名）· qoko（称）。
姓在前，名居中，最后是亲昵称呼。))))
源码 → 渲染效果
哈吉人姓名分三部分：mogu（姓）· naxomi（名）· qoko（称）。姓在前，名居中，最后是亲昵称呼。

适合：整段文字、翻译对照、故事场景。

2. 逐词逐音节 {{ruby: mogu naxomi qoko}}

写法：在文档正文的任意位置，用 {{ruby: 空格分隔的词汇}} 短标记。

这位哈吉人的全名是 {{ruby: mogu naxomi qoko}}（姓·名·称三部分）。
实际渲染成 ↓
这位哈吉人的全名是 mogu naxomi qoko（姓·名·称三部分）。

适合：专有名词、短句、术语解释。

3. 整词一个 ruby {{ruby: <naxomi>}}

写法：在短标记里用 < / > 把词包起来，整个词共享一个 ruby 容器（多个音节排成一行）。

娜笑米（{{ruby: <naxomi>}}）是一位哈吉学者。
实际渲染成 ↓（上方 2 是三个字各自注音；这里三个字共享一条哈吉文）
娜笑米（<naxomi>）是一位哈吉学者。

适合：人名、地名、专有称谓（例如"娜笑米"作为整体呈现，而不是拆成三个汉字）。

提示：以上三种标记可以在同一文档中混用；如果你只是想试一下音节拆分、或想把一整段哈吉语翻译转成 ruby 文本，继续往下看 ↓
